# 教程验证清单

这份清单不是给“写作者自我感动”的，而是给你真正从头到尾走教程时使用的。

它的目标只有一个：

**帮你把卡点、误导点、节奏问题和缺失信息系统记录下来。**

::: tip 使用建议
每一章结束后，只做两件事：

- 勾选是否通过
- 记录真正影响推进的问题

不用一开始就抠措辞，优先记录那些会让你停下来、误解、走偏、或者必须自己脑补的地方。
:::

## 使用说明

状态只填这三个之一：

- `[ ]` 未验证
- `[x]` 已通过
- `[!]` 有问题待修正

问题记录格式：

```md
- 页面：
- 优先级：P0 / P1 / P2 / P3
- 问题：
- 卡点：
- 建议修改：
```

优先级建议：

| 优先级 | 含义 |
|------|------|
| `P0` | 根本无法继续，命令错了、路径错了、逻辑断了 |
| `P1` | 能继续，但明显会误导读者 |
| `P2` | 能继续，但读起来费劲、节奏不好 |
| `P3` | 小优化，表达、排版、提示还能更好 |
