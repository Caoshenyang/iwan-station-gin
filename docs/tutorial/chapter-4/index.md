# 第四章：通用 Web 基础设施

这一章解决的是一个很容易被忽略、但会影响后面所有代码质量的问题：我们的 Web 应用到底该以什么统一方式组织起来。

::: info 🧭 章节衔接
前面三章解决了“项目怎么组织、底座怎么搭好”的问题；这一章继续把 Web 层的公共约定收口。  
等这里的应用启动、参数校验、错误处理和中间件链路都清楚后，第 5 章的认证主线就能写得更稳、更整洁。
:::

## 本章导读

这一章不追求业务功能，而是统一整套后端的 Web 开发范式：
- 应用启动与路由组织
- 请求绑定与参数校验
- 统一响应与错误处理
- 中间件链路设计

## 学习目标

完成本章后，你将：
- ✅ 理解应用入口应该如何组织
- ✅ 知道参数校验应该放在哪一层
- ✅ 建立统一响应与错误处理约定
- ✅ 理清认证、日志、恢复等中间件的执行顺序
