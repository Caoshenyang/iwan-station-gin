# ☕ Java 转 Go 导读

如果你有 Java / Spring 背景，这一页的作用不是教你从零学 Go 语法，而是先帮你建立一张“迁移地图”。

::: tip 💡 怎么使用这页
先用这页建立方向感，不必停留太久。  
你真正理解 Go 工程组织方式，还是要回到正式主线里边做边体会。
:::

## 你最需要先建立的 5 个认知

### 1. Go 没有你熟悉的那套“框架托管感”

在 Spring Boot 里，很多能力来自容器和注解。  
在 Go 里，你会更频繁地自己组织依赖、显式传递上下文、手动串起组件关系。

这不是“更原始”，而是“更直接”。

### 2. Go 更强调组合，而不是继承

你会看到更多：

- `struct`
- interface
- 构造函数
- 显式依赖注入

而不是传统的类继承链和注解驱动扫描。

### 3. 错误处理是显式的

Go 没有 Java 那样的异常流转。  
你会频繁写 `if err != nil`，这会让控制流更明显，但也要求你更主动地组织错误处理。

### 4. 工程结构往往比语法更值得先适应

对 Java 开发者来说，真正的学习成本经常不在 `for range` 或切片，而在这些问题：

- 目录怎么组织
- handler / service / repository 怎么分层
- 中间件放哪
- 配置和依赖怎么串起来

这也是为什么这套教程会把架构和分层放在很靠前的位置。

### 5. 你不是“从零开始学后端”，而是在迁移思维模型

你已经理解：

- HTTP
- REST
- 数据库
- 认证授权
- 分层设计

现在要迁移的是“这些东西在 Go 里应该怎么表达”。
