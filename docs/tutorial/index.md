# 🧭 开始这里

如果你是第一次来到 **Iwan Station Gin**，先不要急着点进某一章写代码。  
这套教程最适合的打开方式，是先建立路线感，再进入正式主线。

::: tip 💡 这页的作用
这不是“课程概述”的重复版，而是整套教程的起点页。读完这页，你应该能回答三件事：

- 这套教程适不适合我
- 我应该走正式主线还是先快速体验
- 我下一步最该点哪一页
:::

## 这套教程适合谁

- 想从零搭建一套 **Gin 后台管理系统** 的开发者
- 有 Java / Spring 背景，想把既有后端经验迁移到 Go 的开发者
- 希望把认证、权限、业务模块、前端对接和部署上线完整走一遍的人
- 不满足于“单点 demo”，想真正理解工程组织方式的人

## 你最终会做出什么

这套教程的目标不是只写几个接口，而是逐步完成一套可以持续扩展的后台项目：

- Go + Gin 后端服务
- JWT 认证与 RBAC 权限
- 内容管理、文件上传、统计接口与缓存
- Vue 3 管理后台
- Docker、Nginx 与生产交付

## 两条阅读路线怎么选

| 路线 | 适合谁 | 入口 |
|------|--------|------|
| **正式主线（推荐）** | 想系统学习、自己从零搭建项目的人 | [课程大纲](./curriculum) |
| **体验路线** | 想先看到最终项目效果，再决定是否深入的人 | [快速开始](./quick-start) |

::: info 📌 默认建议
如果你不确定该怎么选，直接走 **正式主线**。  
这套教程本来就是按“少走弯路”的思路设计的，顺着章节推进会比东跳西跳更省心。
:::

## 当前最推荐的进入方式

### 我想认真学完整套教程

从 [课程大纲](./curriculum) 开始，然后进入 [第一章：环境准备与项目初始化](./chapter-1/)。

### 我想先跑起来看看效果

去 [快速开始](./quick-start)，先把现成项目体验一遍，再回到正式主线。

### 我是 Java 开发者，担心 Go 看不懂

先浏览 [Java 转 Go 导读](./go-for-java-devs)，建立一套对照关系，再进入第一章。

## 学习这套教程的一个小建议

不要把自己当成“来抄答案的人”，而要把自己当成“跟着一条成熟主线做项目的人”。

你不需要一开始就理解全部细节，但要尽量遵循顺序。  
前面章节里很多看起来“有点啰嗦”的设计，往往正是后面少踩坑的关键。
