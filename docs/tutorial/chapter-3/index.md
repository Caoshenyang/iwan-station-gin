# 第三章：配置、日志、数据库与缓存

这一章开始把前面的架构设计真正落成可运行的工程底座。先把配置、日志、数据库和缓存处理干净，后面的认证、权限和业务模块才有稳定落点。

::: info 🧭 章节衔接
上一章我们已经把目录结构、分层架构和依赖关系设计好了；这一章开始把这些设计真正落成可运行的工程骨架。  
学完这一章后，第 4 章的通用 Web 基础设施和第 5 章的认证主线就有了可以承载它们的底座。
:::

## 本章导读

这一章的核心不是“多装几个库”，而是把后面每一章都会反复依赖的底层能力一次搭好：
- 配置管理如何支撑多环境
- 日志系统如何帮助排错和观测
- 数据库连接如何做到稳定可扩展
- Redis 如何作为缓存和运行期能力的基础

## 学习目标

完成本章后，你将：
- ✅ 学会配置管理
- ✅ 集成结构化日志系统
- ✅ 连接 PostgreSQL 数据库（并理解 MySQL 的兼容方式）
- ✅ 集成 Redis 缓存
