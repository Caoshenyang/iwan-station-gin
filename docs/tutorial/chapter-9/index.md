# 第九章：统计接口与缓存优化

这一章聚焦“给前端提供什么数据”和“如何让这些接口在高频访问下依然稳定”。它既是业务模块之后的性能过渡章，也是进入系统管理章节前的重要缓冲。

::: info 🧭 章节衔接
上一章我们已经完成了业务模块和文件能力；这一章开始思考“当系统开始被频繁使用时，接口应该如何更快、更稳”。  
这些统计接口和缓存策略，也会直接成为第 11 章前端看板和首页展示的重要数据来源。
:::

## 本章导读

我们将实现一组面向管理后台的数据接口：
- 统计接口设计
- 后台首页数据建模
- Redis 缓存策略
- 缓存失效与一致性控制

## 学习目标

完成本章后，你将：
- ✅ 设计适合后台首页和图表展示的统计接口
- ✅ 使用 Redis 优化高频读取性能
- ✅ 理解缓存命中、一致性和失效策略的取舍
- ✅ 为后续前端看板展示准备稳定的数据源
