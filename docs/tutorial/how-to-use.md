# 🗺️ 如何使用这套教程

这套教程不是一本“从头读到尾都一样重要”的书，而是一套有主线、有支线、有查阅入口的学习系统。  
如果你按下面的方式使用，整体体验会顺很多。

## 三种使用方式

### 1. 正式主线

这是默认推荐路线，适合第一次完整学习。

你会按章节顺序推进：

1. 环境与项目初始化
2. 项目蓝图与分层架构
3. 配置、日志、数据库与缓存
4. 通用 Web 基础设施
5. 用户体系与 JWT 认证
6. RBAC 权限与菜单模型
7. 内容管理模块
8. 文件上传与对象存储
9. 统计接口与缓存优化
10. 系统配置、审计日志与监控
11. Vue 3 管理后台
12. 测试、安全与发布前检查
13. Docker、Nginx、CI/CD 与生产交付

入口： [课程大纲](./curriculum)

### 2. 体验路线

适合想先看到结果，再决定要不要系统学习的人。

这条路线不会展开解释全部原理，它的目标是让你在更短时间内看到项目全貌。  
体验完成后，仍然建议回到正式主线。

入口： [快速开始](./quick-start)

### 3. 查阅路线

适合已经在主线中推进，或者已经有相关经验，只是回来查某个问题的人。

推荐组合：

- 环境和命令问题： [常见问题](./faq)
- 某个框架或能力点： 参考手册
- Java 概念迁移： [Java 转 Go 导读](./go-for-java-devs)

## 推荐学习顺序

### 如果你是 Go 新手

`开始这里` → `如何使用这套教程` → `课程大纲` → `第一章`

### 如果你是 Java / Spring 转 Go

`开始这里` → `Java 转 Go 导读` → `课程大纲` → `第一章`

### 如果你只想先看项目效果

`快速开始` → `课程大纲` → 回到正式主线补前面章节

## 什么时候不要跳章节

下面这些位置，尽量不要跳过：

- 第一章：环境和初始化决定后面很多命令是否一致
- 第二章：架构和分层决定你是否知道代码应该写在哪
- 第四章：通用 Web 基础设施决定后面接口会不会越写越乱
- 第五、六章：认证和权限是后面业务与前端联调的前置条件

## 什么时候可以回头查

下面这些内容非常适合“先往前走，后面再回来查”：

- Go vs Java 对照
- FAQ
- 参考手册
- 某一章里的扩展说明

## 这套教程默认的一条使用原则

::: tip 🎯 先走主线，再看支线
如果你只记住一条原则，就记这一条。

先保证自己知道“项目是怎么一步步搭起来的”，再去补更多背景、比较和扩展资料。  
这样学到的不是零散技巧，而是一整条可以复用的项目建设路径。
:::
