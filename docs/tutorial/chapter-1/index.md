# 🧭 第一章：环境准备与项目初始化

这一章的目标非常明确：先把开发环境和项目起点处理干净，再进入后面的架构和实现主线。

::: tip 💡 本章怎么读
如果你已经通过「[开始这里](/tutorial/)」和「[课程大纲](/tutorial/curriculum)」建立了整体路线感，这一章就只需要关注两件事：

- 环境是否正确
- 项目初始化是否走在正确轨道上
:::

## 🎯 本章导读

本章将帮助你做好正式进入主线前的准备工作，包括：

- 搭建完整的开发环境
- 初始化项目目录与模块
- 为第二章的架构设计建立稳定起点

## ✅ 学习目标

完成本章后，你将：

- ✅ 搭建好 Go、PostgreSQL、Redis 开发环境
- ✅ 创建项目基础目录结构与模块入口
- ✅ 知道哪些约定会影响后续章节的代码组织

## 🗂️ 章节内容

### 1. [环境搭建](./environment-setup)
- 安装 Go 开发环境
- 配置 PostgreSQL 数据库
- 安装并配置 Redis
- 验证环境是否就绪

### 2. [项目初始化](./project-init)
- 初始化 `go.mod`
- 创建项目基础目录
- 建立最小可运行入口
- 为第二章的架构设计做准备

### 3. [第一次跑通项目](./first-run)
- 整理首次启动需要的最小依赖
- 验证服务能否正常启动
- 建立后续章节可重复验证的起点
